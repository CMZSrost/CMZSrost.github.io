健康码红码标记大数据实践

大数据 spark scala

更新日期:   2023-11-18
文章字数:   3k
阅读次数:  

背景引入

新冠抗疫离不开大数据的支持，在发现感染病人后，可以通过其手机的漫游信息，发现和其行程可能有交集（都在同一个基站注册过）的人员，健康码标注为红色，促使密接人员做核酸筛查。

任务要求

提供漫游信息文件cdinfo.txt与感染号码文件infected.txt,标记所有与感染号码同时段处于同一基站下的手机号码

数据说明

cdinfo.txt （20G）

cdinfo.txt文件为基站下载汇总的人员漫游信息

基站id 时间戳 注册状态 手机号码
55001525849 20220507045223 1 13771874017
55001525849 20220507060705 1 14987849191
55001525849 20220507062150 1 18953076900
55001525849 20220507083519 2 17950088996

基站id：基站的唯一标识
时间戳：手机在基站的注册时间
注册状态：1为进入，2为退出

infected.txt

infected.txt文件为感染的手机号码列表

手机号码
13771874017
14987849191

排名说明

以正确率/完成时长的值大小降序排名

正确率 = 找出的密接人数 / 实际密接人数

完成时长 = 代码提交时间 - 拿到感染人文件时间

数据准备不算时间，可以认为在拿到感染人文件之前，做尽可能多的准备，最后模拟从拿到感染人文件就开始计时，找出所有密接人后停止计时

算法思路

1. 模拟法

采用模拟法对红码进行标记。

1.1 数据清洗

为了获得手机号存在的区间，需要对数据进行清洗，将同一(基站id,手机号)的注册状态一一对应，得到手机号在基站中的区间。

1.2 分组并确定感染区间

对清洗后的数据(基站id,手机号,进入时间,退出时间)按照基站id进行分组，获取感染者名单后，遍历每个分组，获得该分组中所有感染手机号的时间区间并集，得到感染区间。

1.3 标记红码

遍历每个分组，判断该分组中的手机号区间是否与感染区间有交集，若有，则标记为红码。

算法优点

  1. 算法简单，易于理解

算法弊端

  1. 清洗步骤多，确定区间的算法复杂度高，测试周期长
  2. 过早需求感染者名单，有效耗时长
  3. 数据清洗基于数据一进一出的假设，没有考虑到数据可能只有进入或者只有退出的情况，也没有考虑数据多进多出的情况，导致数据清洗不完整不正确，影响正确率

2. 单计数器算法

采用感染者计数器的方法来统计感染区间

2.1 数据分组排序

对数据按照基站id进行分组，每个分组内按照时间戳进行排序，得到(手机号,注册状态,时间戳)的数据集

2.2 计数器统计感染区间

获取感染者名单后，遍历每个分组，遇到感染者则计数器加一，遇到有感染者退出则计数器减一，当计数器大于0时，说明该手机号在感染区间内，标记为红码。

算法优点

  1. 算法简单，易于理解
  2. 算法复杂度低，只需要遍历一次数据
  3. 数据清洗步骤少，测试周期短
  4. 需求感染者名单较晚，有效耗时短

算法弊端

  1. 由于数据分组排序，需要将数据全部加载到内存中，数据量大时，内存不足，导致程序崩溃
  2. 考虑了数据多进多出的情况，但是没有考虑数据只有进入或者只有退出的情况，导致数据清洗不完整不正确，影响正确率

3. 多计数器算法

采用多计数器的方法来统计感染区间

3.1 数据分组排序

对数据按照基站id进行分组，每个分组内按照时间戳进行排序，得到(手机号,注册状态,时间戳)的数据集

3.2 多计数器统计感染区间

获取感染者名单后，为每一个感染者设置一个计数器，遍历每个分组，遇到感染者则对应计数器加一，遇到有感染者退出则对应计数器减一，小于0的计数器时重置为0，当所有计数器之和大于0时，说明该手机号在感染区间内，标记为红码。

算法优点

  1. 算法复杂度低，只需要遍历一次数据
  2. 数据清洗步骤少，测试周期短
  3. 需求感染者名单较晚，有效耗时短
  4. 数据清洗完整，正确率高

算法弊端

  1. 由于数据分组排序，需要将数据全部加载到内存中，数据量大时，内存不足，导致程序崩溃

代码实现与优化

代码说明（多计数器算法）

完整代码放文章末尾

1. 数据预处理

预留sample参数，方便调试时使用小数据集

将"基站id,手机号,时间戳,状态"的字符串数据按照基站id进行分组，得到(基站id,list[(手机号,时间戳,状态)])的数据集

其中状态由1转为1，由2转为-1，方便后续计数器统计

val datapre = sc.textFile(cdinfo)
      .sample(withReplacement = false, samp)
      .map(x=>{
        val line = x.split(",")
        if(line(2).toInt==1)
          (line(0), List((line(3),line(1), 1))) //(基站id,手机号,时间戳,状态)
        else
          (line(0), List((line(3),line(1), -1)))
      })

2. 数据分组排序

将数据按照基站id进行分组，每个分组内按照时间戳进行排序，得到(手机号,注册状态,时间戳)的数据集

val datasort=datapre
  .reduceByKey((x,y)=>x++y) //按基站id分组
  .map(x => x._2
    .toArray
    .sortBy(_._2)
    .map(x=>(x._1,x._3))
    )
  .persist(StorageLevel.MEMORY_AND_DISK_SER)  //persist并设定存储级别以减少操作
datasort.count()

注意事项：

  1. 使用reduceByKey而不是groupByKey，减少shuffle操作，提高效率
  2. toArray将iterator转为array，可能会爆内存
  3. 使用persist并设定存储级别以减少操作，提高效率
  4. 由于RDD类型是惰性操作，需要动作算子激活后才能持久化，因此使用count进行动作操作来持久化数据

3. 感染者计数器

核心算法流程图

val inf = sc.textFile(infected)
val infectC = sc.broadcast(inf.collect())//感染者名单
val cdi = datasort.flatMap(x => { //按基站id遍历
  val infect = infectC.value
  var res: List[String] = List()
  val cnt = Array(0, 0, 0, 0, 0) //设置计数器
  var j = 0
  var k = -1
  for(i<-x.indices){
    k = infect.indexOf(x(i)._1)
    if (k != -1) {
      cnt(k) += x(i)._2
      if(cnt(k)== -1 ){//将计数器小于0的重置为0
        for(h<-j to i){
          res = res++List(x(h)._1)
        }
        j = i
        cnt(k) = 0
      }
    } else if(cnt.sum > 0){
      res = res++List(x(i)._1)
    }
  }
  res
}).distinct()
  .union(inf) //合并，去重
  .sortBy(x=>x,ascending = true,1)
cdi.saveAsTextFile(output) //保存

注意事项：

  1. 使用broadcast广播感染者名单，减少shuffle操作，提高效率
  2. 初始时RDD类型为(基站id,list[(手机号,时间戳,状态)])，遍历时x为list[(手机号,时间戳)]
  3. 使用双指针，指针i正常遍历，当计数器异常时，将异常区间(j,i)的手机号加入结果集，并将指针j移动到异常区间的末尾指针i处
  4. 使用flatMap直接将list结果展开

感想收获

代码编写

代码编写过程中，我首先尝试了数据清洗为时间区间、按基站分组并取交并集得到人员交集的算法，但由于耗时缘故，这种算法并不能满足要求。

接着通过对数据特点的了解与分析、并了解到完成时间的计算规则，去除了数据清洗的内容，尝试在预处理中按时间排序并使用感染者计数器的算法，但是遭遇了排序爆内存的问题。

因此，我向组内分配了搭建集群的任务，同时尝试了kryo序列器以及基于优先队列的分块排序算法，但还是由于技术与时间原因放弃了。

与此同时，集群搭建传来好消息，因此放弃分块排序算法的线路，转向搭建多节点集群的线路。

但是在经过分析后，发现计数器算法会受到只出不进情况的影响，因此加上了数据清洗流程，增加了任务的测试周期。

后来在与组员、其他组成员讨论并在网上学习后，成功对计数器算法进行了改良，使其能应对只进不出情况的影响，并免去了数据清洗的过程。经过修饰，形成了最终的代码，即非数据清洗的双指针遍历算法。

集群搭建

搭建集群过程离不开与大家的协作。

最初搭建集群遭遇了nat网卡ipv4不存在、HostOnly网卡ipv4相同等问题，第一次搭建集群失败。

第二次提前搭建了联通ip与ssh的虚拟机，经过简单修饰后成功在机房ping通。

接着与搭建集群的同学一起协作，成功搭建起spark集群并提交通过采样率0.05的jar包。

最后在第三次搭建集群中提交运行jar包并录像。

收获

这次大作业我学到了许多东西，不仅熟悉了spark中RDD相关算子，也了解了持久化，自定义分区器、闭包、聚合算子性能区别、序列化、kryo序列器、spark三类部署方法等相关知识并做了相应的实践，积累了经验。同时这次大作业也加强了我管理团队，把控任务进度的能力。

package Demo

import org.apache.spark.storage.StorageLevel
import org.apache.spark.{SparkConf, SparkContext}


object FindRed {
  def main(args: Array[String]): Unit = {
    var conf = new SparkConf().setAppName("FindRed")
    var samp = 0.05
    if(args.length==4){conf = conf.setMaster("local");samp=1}
    val sc = new SparkContext(conf)
    sc.setLogLevel("ERROR")
    val starttime: Long = System.currentTimeMillis()
    // RDD处理
    val infected = args(0)
    val cdinfo = args(1)
    val output = args(2)
    val datapre = sc.textFile(cdinfo)
      .sample(withReplacement = false, samp)
      .map(x=>{
        val line = x.split(",")
        if(line(2).toInt==1)
          (line(0), List((line(3),line(1), 1))) //(基站id,手机号,时间戳,状态)
        else
          (line(0), List((line(3),line(1), -1)))
      })
      .reduceByKey((x,y)=>x++y) //按基站id分组
      .map(x => x._2
        .toArray
        .sortBy(_._2)
        .map(x=>(x._1,x._3))
        )
      .persist(StorageLevel.MEMORY_AND_DISK_SER)  //persist并设定存储级别以减少操作
    datapre.count() //进行动作操作来持久化数据，由于还没拿到感染数据，所以不算时间
    //拿到感染数据
    val pretime: Long = System.currentTimeMillis()
    println("----数据预处理完毕----")
    val inf = sc.textFile(infected)
    val infectC = sc.broadcast(inf.collect())//感染者名单
    val cdi = datapre.flatMap(x => { //按基站id遍历
      val infect = infectC.value
      var res: List[String] = List()
      val cnt = Array(0, 0, 0, 0, 0)
      var j = 0
      var k = -1
      for(i<-x.indices){
        k = infect.indexOf(x(i)._1)
        if (k != -1) {
          cnt(k) += x(i)._2
          if(cnt(k)== -1 ){
            for(h<-j to i){
              res = res++List(x(h)._1)
            }
            j = i
            cnt(k) = 0
          }
        } else if(cnt.sum > 0){
          res = res++List(x(i)._1)
        }
      }
      res
    }).distinct()
      .union(inf) //合并，去重
      .sortBy(x=>x,ascending = true,1)
    cdi.saveAsTextFile(output) //保存
    val endtime: Long = System.currentTimeMillis()
    println("红码名单数量:\t", cdi.collect.length)
    println("数据预处理耗时:\t", (pretime - starttime) / 1000, "s")
    println("总计标记红码时长:\t", (endtime - pretime) / 1000, " s")
    println("总计时长:\t", (endtime - starttime) / 1000, " s")
  }
}
